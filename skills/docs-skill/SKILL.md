# Docs Skill

Documentation detection and generation for code changes.

## Role
`docs`

## Description
The Docs Skill enables agents to identify undocumented code changes and automatically generate or suggest documentation updates. This includes README snippets, API documentation, and usage examples.

## Capabilities
- Detect undocumented code changes
- Generate documentation snippets
- Validate documentation completeness
- Suggest README updates
- Create API documentation
- Generate usage examples
- Maintain documentation index
- Archive deprecated docs

## Related Playbooks
Implementation details: [gitmesh/playbooks/docs](../playbooks/docs)

## Configuration
Enabled by default when creating a Docs Agent via `gitmesh-agents setup`.
